Changes Sidebar — Architecture Approaches

How to structure the right sidebar within the existing NavigationSplitView layout

A

HSplitView Inside Detail ★ Recommended

Wrap the session detail area in an HStack with a ResizableDivider and the changes panel. The right sidebar lives inside the NavigationSplitView detail column.

Pros

  • Uses existing ResizableDivider component
  • Sidebar state is per-session (natural)
  • No changes to NavigationSplitView structure
  • Consistent with how the left sidebar already works

Cons

  • Detail column needs layout rework
  • Animation must be manually implemented
B

NavigationSplitView 3-Column

Switch from 2-column to 3-column NavigationSplitView (sidebar / content / supplementary). The changes panel becomes the third column.

Pros

  • Native macOS inspector/supplementary column pattern
  • Built-in collapse/expand behavior
  • System-consistent animations

Cons

  • 3-column NavigationSplitView has rigid width constraints
  • Supplementary column always on right — can't scope to sessions only
  • Major refactor of the window layout
  • Column visibility API is limited in SwiftUI
C

Overlay Panel with Animation

The changes panel slides in from the right as an overlay on top of the detail view (like Xcode's inspectors), without resizing the terminal.

Pros

  • Terminal doesn't reflow when panel opens
  • Simple slide-in/out animation
  • No layout restructuring needed

Cons

  • Obscures terminal content underneath
  • Not resizable (fixed width or percentage)
  • Feels less native — more like a mobile pattern
